## ✅ テストシナリオ検証

### Task 1: マッチなしパターン
**目的**: CSV側に対応するレコードがない場合の処理確認

**実装内容**:
- 抽出結果に存在しない daily_report_no/tablet_no を追加
- 複合キー（日付+店舗名+氏名）でも一致なし

**検証結果**: ✅ **成功**
- matching_key: "見つかりませんでした"
- status: "要確認"
- review_reason: "CSV内に対応するレコードが見つかりません"

---

### Task 2-a: 複数候補＆DataNo第1優先パターン
**目的**: 複数候補が存在しても DataNo で第1優先確定される動作確認

**実装内容**:
- Salesforce CSV に同じ date+store_name+staff_name を持つレコード2件追加（DataNo異なる）
- 帳票読み取り結果側の Report 1 は一致する DataNo を持つ

**検証結果**: ✅ **成功**
- matching_key: "日報DataNo"（複数候補存在下でも第1優先確定）
- status: "一致"（全項目完全一致）
- 複合キーマッチングに進まない（第1優先により確定）

---

### Task 2-b: 複合キーマッチング＆複数候補パターン
**目的**: 複合キーで複数候補が出た場合、自動確定せず要確認になること確認

**実装内容**:
- 帳票読み取り結果側: DataNo/TabNo が CSV に存在しないが、date+store_name+staff_name は一致するデータを追加
- Salesforce CSV側: Task 2-a の複数候補（3件）に加え、さらに重複行を追加

**検証結果**: ✅ **成功**
- matching_key: "複合キー（複数候補）"
- status: "要確認"（複数候補のため自動確定しない）
- review_reason: "複数の候補レコードが見つかりました（3件）"
