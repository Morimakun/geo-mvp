## 📌 重要な発見

### 1. Crop座標の重要性
- **実帳票との差分**: PDFサイズ 6% 縮小にもかかわらず、ratio ベースだけでは対応不可
- **テキスト位置分析**: PyMuPDF の get_text("dict") で正確な位置を特定可能
- **調整幅**: left_totals top 0.78→0.72（-0.06）、right_totals top 0.55→0.43（-0.12）

### 2. Vision API の読取精度
- **高精度項目**: 日付、店舗名、氏名（フォントサイズ大、背景クリア）→ 100%
- **中精度項目**: 日報DataNo、タブレットNo（小フォント、背景パターンあり）→ 100% 読取、一部特殊文字誤読
- **低精度項目（調整前）**: 合計欄（手書き、小面積、背景ノイズ）→ 0-6%
- **低精度項目（調整後）**: 合計欄（crop精度向上により）→ 100%

**結論**: Crop精度がVision API精度の上限を決定

### 3. 特殊文字の読み誤り
タブレットNo で以下の誤読を確認：
- 「it」→ 「¶t」（1件）
- 「it -」→ 「9t -」（1件）

**対策**: CSV照合時に曖昧性許容度を設定するか、正規化処理を追加

## 🎯 次フェーズへの推奨事項

### Phase 4: CSV照合ロジックの実装（次ステップ）

**要件**:
1. **照合キーの優先順** (既定):
   - 第1優先: 日報DataNo（完全一致）
   - 第2優先: タブレットNo（完全一致、または正規化後一致）
   - 第3優先: 日付 + 店舗名 + 氏名（複合キー）

2. **ステータス判定**:
   - **一致**: キー照合OK かつ 合計欄の差分なし
   - **不一致**: キー照合OK かつ 合計欄に差分あり
   - **要確認**: キー照合不可 OR 複数マッチ

3. **特殊文字の正規化**:
   - タブレットNo の「it - 」「it-」「¶t -」を同一視
   - または、推測せず「要確認」に分類
